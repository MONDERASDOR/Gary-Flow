Why Gary Flow?

Multi-Version Support: Seamlessly proxy Minecraft clients from 1.7 to 1.20+ with real protocol translation.
Extreme Performance: C core with async IO, zero-copy, and optimized concurrency for 10,000+ players.
Universal Plugins: Java plugin API for proxy and Paper/Bungee events, hot-reload, and full extensibility.
Horizontal Scaling: Backend server pools, load balancing, and failover for massive Minecraft networks.
Security & Anti-Bot: Built-in DDoS protection, IP filtering, and advanced connection management.
Open Source: Free, transparent, and community-driven. No artificial limits, no TODOs, no compromises.

Getting Started

  1. Build the Proxy:
    ./scripts/build_all.sh (Linux/macOS) or scripts\build_all.bat (Windows)
  2. Configure Backend Servers:
    Set up Paper servers with online-mode=false and unique ports.
  3. Edit Gary Flow Config:
    List backend servers in config.yml (see docs).
  4. Run Gary Flow:
    ./build/garyflow (Linux/macOS) or build\garyflow.exe (Windows)
  5. Connect:
    Players join via Gary Flow’s IP/port (default: 25565).

Scaling & Performance

  • Async IO (epoll/IOCP), non-blocking sockets, thread pools
  • Efficient memory/session management, zero-copy packet handling
  • Horizontal scaling: add more backend servers as needed
  • No artificial player limits—hardware is the only constraint
  • Use a modern multi-core CPU, 32GB+ RAM, high-bandwidth network

Plugin System

  • Java plugin API: listen to all major proxy events (join, quit, chat, switch, etc)
  • Hot-reload, dynamic loading/unloading, plugin isolation
  • Paper/Bungee plugin compatibility
  • Real-time event dispatch via JNI bridge
  • Place plugins in java/plugins/
